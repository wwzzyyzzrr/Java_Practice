Less守护命名空间

当保护被施加到命名空间，在命名空间中定义混合类型只用于保护条件返回 true 时。命名空间防护类似防护混合类型。

示例

下面的例子演示了 Less 文件中使用守护命名空间： 
<html>
<head>
  <link rel="stylesheet" href="style.css" type="text/css" />
  <title>Guarded Namespaces</title>
</head>
<body>
<h2>Welcome to Yiibai Yiibai</h2>
<p>This will paragraph be displayed red, when (@color = blue) in style.less and when color is other than blue, then this paragraph will be default black.</p>
</body>
</html>
接下来，创建文件 style.less。

style.less 

@import "http://www.yiibai.com/less/lib.less";
#namespace when (@color = blue) {
  .mixin() {
   color: red;
  }
}
p{
 #namespace .mixin();
}
这将在从下面代码的路径  http://www.yiibai.com/less/lib.less 导入所述lib.less文件到 style.less ：

lib.less 

@color: blue;
你可以编译 style.less 使用下面的命令来生成 style.css 文件： 
lessc style.less style.css
接着执行上面的命令，它会自动创建 style.css 文件，下面的代码：

style.css 

p {
  color: red;
}
