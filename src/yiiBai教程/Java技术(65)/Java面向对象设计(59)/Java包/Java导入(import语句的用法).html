Java导入(import语句的用法)

导入(import）声明用于将任何类型导入编译单元。导入(import）声明出现在包声明之后，第一个类型声明之前。

有两种类型的导入声明：

  • 单类型导入声明
  • 按需导入声明

单类型导入声明

单类型导入声明用于从包导入单个类型。 它的形式如下 - 

import <fully qualified name of a type>;

以下导入声明从com.yiibai包导入Dog类：

import com.yiibai.Dog;

单一类型的导入声明只从包中导入一个类型（一个类）。要从包中导入多个类(或型类)，请为每个类型使用单独的导入声明。

以下导入声明从pkg1包导入ClassOne，从pkg2包导入ClassTwo和ClassThree，以及从pkg3包导入ClassFour，如以下代码：

import pkg1.ClassOne; 
import pkg2.ClassTwo; 
import pkg2.ClassThree; 
import pkg3.ClassFour;

以下代码使用Dog类的完全限定名称。

public  class  Main{
   public static  void  main(String[]  args)  {
      com.yiibai.Dog jack;  // Uses  full qualified name for the   Dog  class
   }
}

修改后的Main类声明如下：

import com.yiibai.Dog; // Import   the   Dog  class

public  class  Main {
   public static  void  main(String[]  args)  {
      Dog  jack; // Use simple name of  the   Dog  class
   }
}

当编译器在语句中遇到Dog类的简单名称时，如以下声明 -

Dog  jack;

它将通过所有导入声明将简单名称解析为完全限定名称。当它尝试解析简单名称Dog时，它会找到导入声明import com.java2s.Dog 并将导入Dog类。

在上面的语句中使用简单名称Dog时，实际指定使用com.yiibai.Dog类。编译器用以下语句替换上面的语句：

com.yiibai.Dog jack;

导入声明允许在代码中使用类型的简单名称，从而使代码更具可读性。当编译代码时，编译器用其完全限定名替换类型的简单名称。它使用导入声明将类型的简单名称转换为其完全限定名称。

按需导入声明

按需导入声明使用一个导入声明从包导入多个类型。按需导入声明的语法是 - 

import <package name>.*;

这里，包名称后面跟着一个点（.）和一个星号（*）。例如，以下按需导入声明从 com.yiibai 包导入所有类型：

import com.yiibai.*;

可以使用按需导入声明重写Main类的代码。

import com.yiibai.*;

public  class  Main {
    public static  void  main(String[]  args)  {
        Dog  jack; // Use simple name of  the   Dog  class
    }
}

静态导入声明

静态导入声明将类型的静态成员（变量/方法）导入到编译单元中。静态导入声明有两种风格：单静态导入和按需静态导入。
单静态导入声明从类型中导入一个静态成员。 按需静态导入声明导入类型的所有静态成员。
静态导入声明的一般语法如下：

  • 单静态import语句：
import static <package  name>.<type name>.<static member name>;
  • 按需静态导入声明-import语句：
    import static <package  name>>.<type  name>.*;
    System是java.lang包中的一个类，它有一个名为out的静态变量。
    当使用System.out时，指的是System类中的静态变量。可以使用静态导入声明从System类导入out静态变量，如下所示：
import static  java.lang.System.out;

以下代码导入System类的out静态变量。

import static  java.lang.System.out;

public class  StaticImportTest  {
    public static  void  main(String[]  args)  {
        out.println("Hello static  import!");
    }
}

上面的代码生成以下结果。

Hello static  import!

java.lang包中的Math类有许多实用程序常量和静态方法。
例如，它有一个类变量名为PI，就是那个圆周率。要使用Math类中的任何静态变量或方法，则需要使用类名Math来限定它们。
例如，可以将PI静态变量称为Math.PI，将sqrt（）方法称为Math.sqrt（）。可以使用以下按需静态导入声明来导入Math类的所有静态成员：

import static  java.lang.Math.*;

实例

以下代码通过导入其静态成员来演示使用Math类。参考如下代码 - 

import static java.lang.Math.PI;
import static java.lang.Math.sqrt;
import static java.lang.System.out;

public class Main {
  public static void main(String[] args) {
    double radius = 9.9;
    double area = PI * radius * radius;

    out.println("Value of  PI  is: " + PI);
    out.println("Radius of  circle: " + radius);
    out.println("Area of  circle: " + area);
    out.println("Square root of  8.0: " + sqrt(8.0));
  }
}

上面的代码生成以下结果。

Value of  PI  is: 3.141592653589793
Radius of  circle: 9.9
Area of  circle: 307.90749597833565
Square root of  8.0: 2.8284271247461903

以下是关于静态导入声明的一些重要规则。

  • 如果导入两个具有相同简单名称的静态成员，一个使用单静态导入声明，另一个使用静态按需导入声明，则使用单静态import声明导入的静态成员优先。
  • 使用单静态导入声明导入具有相同简单名称的两个静态成员是不允许的。
  • 如果使用单静态导入声明导入静态成员，并且在同一类中存在具有相同名称的静态成员，则使用类中的静态成员。
