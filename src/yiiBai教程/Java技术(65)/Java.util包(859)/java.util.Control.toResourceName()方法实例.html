java.util.Control.toResourceName()方法实例

java.util.ResourceBundle.Control.toResourceName(String bundleName, String suffix) 方法给定bundleName转换为通过替换所有出现所需的ClassLoader.getResource方法的形式'.'在bundleName以'/'，并附加一个'.'和给定的文件后缀。例如，如果bundleName为“foo.bar.MyResources_ja_JP”和后缀是“properties”，那么“foo/bar/MyResources_ja_JP.properties”返回。

声明

以下是java.util.Control.toResourceName()方法的声明

public final String toResourceName(String bundleName, String suffix)

参数

  • baseName -- 软件包名称

  • suffix -- 文件类型的后缀

返回值

此方法返回转换后的资源名称

异常

  • NullPointerException -- 如果bundleName 或suffix 为 null

例子

下面的示例演示java.util.ResourceBundle.Control.toResourceName()方法的用法。

package com.yiibai;

import java.util.Locale;
import java.util.ResourceBundle;
import java.util.ResourceBundle.Control;

public class ResourceBundleControlDemo {

   public static void main(String[] args) {

      // create a new ResourceBundle.Control with default format
      ResourceBundle.Control rbc =
              ResourceBundle.Control.getControl(Control.FORMAT_DEFAULT);

      // print the name of the resource
      System.out.println("" + rbc.toResourceName("hello", "properties"));

   }
}

假设在你的CLASSPATH中的资源文件hello_en_US.properties可用，包含以下内容。该文件将被用作输入的示例程序：

hello=Hello World!

让我们来编译和运行上面的程序，这将产生以下结果：

hello.properties
