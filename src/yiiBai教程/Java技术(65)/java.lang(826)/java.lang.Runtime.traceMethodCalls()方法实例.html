java.lang.Runtime.traceMethodCalls()方法实例

java.lang.Runtime.traceMethodCalls(boolean on) 方法，使产生的方法调用/禁用跟踪。如果布尔参数为true，则此方法意味着Java虚拟机发出调试，因为它是所谓的调试信息中虚拟机的每个方法。该信息的格式，以及它所发出的文件或其它输出流，取决于主机环境。虚拟机可能会忽略这个要求，如果它不支持此功能。调用此方法与参数错误提示虚拟机停止启动的调试信息。

声明

以下是java.lang.Runtime.traceMethodCalls()方法的声明

public void traceMethodCalls(boolean on)

参数

  • on -- 设置true使指令跟踪; 设置为false以禁用此功能。

返回值

这个方法没有返回值。

异常

  • NA

例子

下面的例子显示lang.Runtime.traceMethodCalls()方法的使用。

package com.yiibai;

public class RuntimeDemo {

   public static void main(String[] args) {

      // print the state of the program
      System.out.println("Program is starting...");

      // start tracing for instructions
      System.out.println("Enabling tracing...");
      Runtime.getRuntime().traceMethodCalls(true);
      System.out.println("Done!");
   }
}

让我们来编译和运行上面的程序，这将产生以下结果：

Program is starting...
Enabling tracing...
Done!
